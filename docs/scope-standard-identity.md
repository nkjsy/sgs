# 范围约束：标准版身份场（当前阶段）

更新时间：2026-02-22

## 当前硬约束

本项目当前阶段固定为：

- 规则版本：标准版（standard）
- 对局模式：身份场（identity）
- 武将范围：标准版武将池（standard-only，待武将系统上线后执行）

补充口径说明：

- 当前牌堆实现采用“标准身份场 108 张（标准 + EX 并入）”口径。
- 明确不引入军争、国战、SP 扩展牌与对应规则链。

对应代码常量：

- `packages/core/src/standard-scope.ts` 中的 `CURRENT_SCOPE`
- `packages/core/src/standard-scope.ts` 中的 `CURRENT_STANDARD_IDENTITY_CARD_KINDS`

## 已执行的防跑偏措施

1. 牌堆构建时做范围校验：
   - `createDeck()` 会校验每个牌种是否属于当前“标准版身份场”允许集合。
   - 若误加入超范围牌种，会抛出错误并阻断构建流程。
2. 文档口径统一：
   - 牌堆差异与规则覆盖均以“标准版身份场 MVP”描述，避免混入军争/国战扩展语义。

## 预留扩展余地（不破坏当前约束）

后续扩展建议按“新增配置”而非“覆盖当前配置”进行：

1. 在 `standard-scope.ts` 旁新增扩展范围常量（例如 `MILITARY_STRUGGLE_SCOPE`、`NATIONAL_WAR_SCOPE`）。
2. 为牌堆工厂引入显式参数（例如 `profile`），按范围选择不同牌种集合。
3. 武将系统上线时，引入“将池 profile”筛选层，默认仍锁定 `standard-only`。
4. 新增规则时序时，先保证标准身份场回归，再启用扩展模式测试。

## 本阶段目标边界

- 只继续补齐“标准版身份场”缺失牌与规则。
- 不引入军争、国战、SP 专属牌与对应时序。
- 不引入非标准武将池。
