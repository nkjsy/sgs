# SGS 单机版（TypeScript）

这是一个三国杀单机 MVP：1 名玩家位 + 4 名 AI，先实现规则正确优先。

## 项目结构

- `packages/core`：规则内核与 AI 决策接口。
- `apps/single`：本地对局驱动（命令行模拟，按阶段/出牌/响应/判定等分层输出日志）。
- `apps/web`：Web 交互层 MVP（场面、手牌、动作按钮、事件日志，复用 core 状态机推进）。

## 当前进度

- 已实现：回合流程（含判定阶段）、基础牌（杀/闪/桃）、基础伤害与濒死处理、5 人身份配置、基础 AI、出牌阶段【杀】限一次、距离/攻击范围（含武器范围与+1/-1坐骑修正）、【顺手牵羊】【过河拆桥】【决斗】【南蛮入侵】【万箭齐发】【桃园结义】【五谷丰登】【借刀杀人】【无中生有】与【无懈可击】响应链（MVP 简化版，含桃园/五谷按目标可被无懈抵消、借刀可无懈且强制杀可被闪）、延时锦囊【乐不思蜀】【闪电】（MVP 简化版），标准武将技能系统（标准将池技能已接入规则层）以及标准身份场 108 口径牌堆（含花色点数）。
- Web 交互新增：人类响应窗口已覆盖打闪/出杀/无懈/救援（含急救）及护驾/激将代打确认；无懈支持逐次确认后统一结算。
- 未实现：完整卡池与扩展模式（军争/国战等）、完整官方复杂时序细则、完整 Web 美术化与联机体验（当前为可玩 MVP）。

## 官方规则对照

本项目当前并未“完整实现官方规则集”。

- 已按官方规则实现的最小子集：
	- 回合主流程（判定/摸牌/出牌/弃牌/结束）。
	- 基本牌中的【杀】【闪】【桃】核心效果。
	- 锦囊中的【桃园结义】【五谷丰登】基础效果。
	- 出牌阶段【杀】额定次数上限为 1。
	- 基础濒死与死亡处理。
	- 延时锦囊基础机制（乐不思蜀、闪电传递与命中伤害）。
	- Web 响应窗口（MVP）：已覆盖闪/杀/无懈/桃（含急救）与护驾/激将代打确认；无懈支持逐次确认后结算。

- 尚未实现的关键规则：
	- 锦囊牌与装备牌完整结算（当前为“已实现牌种 + 部分技能细则”状态）。
	- 高级时序与完整官方细则（判定区仅有延时锦囊最小子集，复杂优先级与响应链未完整实现）。

详细差异见 [docs/rule-coverage.md](docs/rule-coverage.md)。

牌堆完整度对照见 [docs/official-deck-gap.md](docs/official-deck-gap.md)。

## 借刀杀人边界清单（已测）

以下条目已由 `packages/core/src/test/engine.test.ts` 回归用例覆盖：

- 合法性与响应链
	- 可被【无懈可击】抵消。
	- 强制出的【杀】可被【闪】响应。
	- 非法目标（不满足持刀者攻击范围）不会消耗借刀牌。
- 失败分支
	- 持刀者无法打出【杀】时，武器转移给使用者。
- 与武器技能联动
	- 青龙偃月刀：被闪后可追击；无后续【杀】时终止。
	- 丈八蛇矛：可两牌转化出【杀】；有实体【杀】时优先实体【杀】；不足两牌且无【杀】时无法出杀。
	- 青釭剑 × 八卦阵：青釭无视八卦判定；非青釭武器下八卦可正常判定闪避。
	- 仁王盾：黑色【杀】无效，红色【杀】可命中。
	- 贯石斧：被闪后可弃两牌令【杀】仍命中。
	- 寒冰剑：可防止本次伤害并弃置目标两张牌。
	- 麒麟弓：造成伤害后可弃置目标坐骑。
	- 方天画戟：借刀强制杀按借刀指定目标结算，不自动追加额外目标。

## 运行

1. 安装依赖：`npm install`
2. 构建：`npm run build`
3. 启动单机模拟：`npm run start:single`
4. 启动 Web 交互版：`npm run start:web`

## 初始化配置（core）

`createInitialGame(seed, options?)` 支持以下可选参数：

- `nullifyResponsePolicy`：无懈响应优先级策略
	- `camp-first`（默认）：身份同阵营优先
	- `seat-order`：按座次优先

示例：

- 默认策略：`createInitialGame(42)`
- 座次优先：`createInitialGame(42, { nullifyResponsePolicy: "seat-order" })`
