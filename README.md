# SGS 单机版（TypeScript）

这是一个三国杀单机 MVP：1 名玩家位 + 4 名 AI，先实现规则正确优先。

## 项目结构

- `packages/core`：规则内核与 AI 决策接口。
- `apps/single`：本地对局驱动（命令行模拟，按阶段/出牌/响应/判定等分层输出日志）。
- `apps/web`：Web 展示层骨架（后续接入完整交互）。

## 当前进度

- 已实现：回合流程（含判定阶段）、基础牌（杀/闪/桃）、基础伤害与濒死处理、5 人身份配置、基础 AI、出牌阶段【杀】限一次、距离/攻击范围（含武器与+1/-1坐骑修正）、【顺手牵羊】【过河拆桥】【决斗】【南蛮入侵】【万箭齐发】【桃园结义】【五谷丰登】【借刀杀人】与【无懈可击】响应链（MVP 简化版，含桃园/五谷按目标可被无懈抵消、借刀可无懈且强制杀可被闪）、延时锦囊【乐不思蜀】【闪电】（MVP 简化版）。
- 未实现：完整卡池、武将技能系统、完整 Web 交互。

## 官方规则对照

本项目当前并未“完整实现官方规则集”。

- 已按官方规则实现的最小子集：
	- 回合主流程（判定/摸牌/出牌/弃牌/结束）。
	- 基本牌中的【杀】【闪】【桃】核心效果。
	- 锦囊中的【桃园结义】【五谷丰登】基础效果。
	- 出牌阶段【杀】额定次数上限为 1。
	- 基础濒死与死亡处理。
	- 延时锦囊基础机制（乐不思蜀、闪电传递与命中伤害）。
- 尚未实现的关键规则：
	- 锦囊牌与装备牌完整结算（当前仅实现部分锦囊与 3 张装备牌的简化逻辑）。
	- 武将技能与高级时序（判定区仅有延时锦囊最小子集，复杂优先级与响应链未完整实现）。

详细差异见 [docs/rule-coverage.md](docs/rule-coverage.md)。

## 运行

1. 安装依赖：`npm install`
2. 构建：`npm run build`
3. 启动单机模拟：`npm run start:single`

## 说明

首版重点是可读性与可维护性：导出类型与关键函数均有文档注释，便于后续扩展武将、牌和 AI。
